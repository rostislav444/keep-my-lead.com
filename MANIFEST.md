# LeadBot — Product Manifest
> Версия 0.2 | Март 2026

---

## Цель

LeadBot — мультитенантная SaaS-платформа для автоматической обработки лидов из Instagram.
Когда потенциальный клиент комментирует рекламный пост или пишет в Direct,
бот мгновенно вступает в диалог, квалифицирует интерес, прогревает —
и передаёт **горячего лида** менеджеру для звонка или закрытия сделки.

Бот не просто отвечает на вопросы. Бот **продаёт**.

**Проблема которую решаем:**
Малый бизнес теряет до 60% лидов из Instagram из-за медленных ответов.
Менеджеры не успевают, работают в разное время, забывают.
Каждый потерянный лид — это деньги выброшенные на рекламу.

**Для кого:**
Эксперты, онлайн-школы, косметологи, фитнес-тренеры, юристы —
все кто продаёт услуги или курсы через Instagram.

|---|---|
| Скорость | Ответ в течение секунд, 24/7 |

| Масштаб | Один бот = 1000 менеджеров одновременно |

## Мультитенантность

LeadBot — это **платформа**, не инструмент для одного бизнеса.

```
LeadBot Platform
├── Компания A (косметологическая школа)
│   ├── Instagram аккаунт: @beauty_school
│   ├── Каталог: курс "Перманентный макияж", курс "Брови"
│   └── Бот настроен под их стиль и продукты
│
├── Компания B (фитнес-тренер)
│   ├── Instagram аккаунт: @fit_coach
│   ├── Каталог: онлайн-программа, персональные тренировки
│   └── Бот настроен под их стиль и продукты
│
└── Компания C (юридические услуги)
    ├── Instagram аккаунт: @lawyer_ua
    ├── Каталог: консультация, сопровождение сделок
    └── Бот настроен под их стиль и продукты
```

Каждая компания — изолированный тенант со своим:
- Instagram аккаунтом (access token)
- Каталогом продуктов
- Настройками бота
- Базой лидов и диалогов

## Модули

### 1. Auth & Onboarding
- Регистрация компании (тенанта) — название, сфера, контактное лицо
- OAuth Flow: подключение Instagram Business аккаунта через Meta API
- Сохранение access token per tenant
- Один тенант может подключить несколько Instagram аккаунтов

### 2. Product & Service Catalog

Это **сердце** контекста бота. Каждая услуга/продукт содержит:

```
Product
├── name              — название (напр. "Курс перманентного макияжа")
├── type              — услуга / курс / товар / консультация
├── short_description — 1-2 предложения для быстрого ответа
├── full_description  — полное описание для глубокого диалога
├── price             — цена (или "от X")
├── price_context     — что входит в цену, рассрочка, скидки
├── value_points      — список ключевых ценностей продукта
│                       (напр. ["17 лет опыта", "сертификат", "практика на моделях"])
├── objection_answers — заготовленные ответы на типичные возражения
│                       ("дорого", "подумаю", "а у вас есть опыт?")
├── target_audience   — для кого продукт (портрет клиента)
├── process           — как проходит покупка/обучение (шаги)
├── result            — что получит клиент в итоге
├── format            — онлайн/оффлайн, длительность, формат занятий
├── faq               — часто задаваемые вопросы и ответы
└── payment_link      — ссылка на оплату / запись
```

Всё это автоматически компилируется в системный промпт для Claude.

### 3. Bot Configuration

Настройки бота per tenant:
- **Тон общения** — дружелюбный, профессиональный, экспертный
- **Имя бота** — как представляться (напр. "Я Алина, помощник школы Beauty Expert")
- **Цель диалога** — получить имя + номер телефона для звонка менеджера
- **Триггер эскалации** — когда передавать живому менеджеру
- **Запрещённые темы** — что бот не обсуждает

### 4. Bot Engine — Логика продаж

Бот работает по воронке, не просто отвечает:

```
Шаг 1: ЗАХВАТ
  Человек комментирует → бот пишет в DM
  "Привет! Вижу вас интересует [тема поста]. Расскажу подробнее?"

Шаг 2: КВАЛИФИКАЦИЯ
  Бот выясняет: есть ли опыт, какая цель, когда планирует начать

Шаг 3: ПРЕЗЕНТАЦИЯ
  На основе каталога рассказывает о продукте через ценности и результат
  Работает с возражениями используя objection_answers

Шаг 4: СБОР КОНТАКТА
  "Чтобы ответить на все вопросы, наш менеджер свяжется с вами.
   Как вас зовут и на какой номер позвонить?"

Шаг 5: ПЕРЕДАЧА ЛИДА
  Бот сохраняет лид, уведомляет менеджера в Telegram
  Диалог помечается как "горячий"
```

### 5. Dialog Manager
- История каждого диалога с метаданными
- Статусы: новый → в диалоге → квалифицирован → лид → закрыт
- Ручной режим: менеджер перехватывает диалог одной кнопкой
- Автопауза бота когда менеджер подключился

### 6. Lead Manager
- Карточка лида: имя, телефон, интерес, продукт, источник
- Уведомление менеджера (Telegram бот)
- Экспорт лидов в CSV

### 7. Dashboard
- Список всех диалогов с фильтрами по статусу
- Горячие лиды — отдельный раздел, требующий внимания
- Метрики: диалоги, конверсия в лид, среднее время до контакта

## Модели базы данных

```python
Tenant          # компания-клиент LeadBot
InstagramAccount # подключённый аккаунт (belongs to Tenant)
Product         # продукт/услуга (belongs to Tenant)
ValuePoint      # ключевые ценности продукта
ObjectionAnswer # ответы на возражения
FAQ             # вопросы и ответы по продукту
BotConfig       # настройки бота (belongs to Tenant)
Dialog          # диалог с конкретным Instagram пользователем
Message         # сообщение в диалоге
Lead            # квалифицированный лид (имя + телефон + интерес)
```

## MVP — что входит в v1.0

- [ ] Мультитенантность: регистрация компаний, изоляция данных

**Не входит в v1.0:**
- WhatsApp интеграция
- A/B тестирование промптов
- Детальная аналитика
- White-label
- Экспорт в CRM
